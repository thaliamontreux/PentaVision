Blocklist Integration

Guidance for consuming the blocklist feed and hardening access to port 7080.

pfSense / firewall consumption

Use the CSV endpoint as the single source of truth for downstream enforcement.

CSV URL
{{ csv_url }}
Recommended polling
Every 30–60 seconds (tune to your environment). The response is deterministic and uses short TTL no-cache headers.
Token auth (optional)
If enabled, consumers must send:
Authorization: Bearer <token>
Firewall usage pattern
Create an alias/table fed by the URL, then reference that alias in WAN rules (block) and optionally IDS/IPS correlation.

Hardening & network controls

Recommended controls so port 7080 never becomes an attack vector.

Consumer IP allowlist
Set this in Admin → Blocklist Distribution. Only the listed firewall/IDS IPs should be able to reach 7080.
UFW example
Allow only your firewall subnet to reach 7080:
sudo ufw allow from 192.168.250.0/24 to any port 7080 proto tcp
mTLS option (recommended for high assurance)
Terminate TLS with client certificate authentication in front of port 7080 (Apache or Nginx reverse proxy). Keep the service itself read-only.
Example (Apache, sketch): enable ssl and proxy, then require client certs:
<VirtualHost *:443> ServerName blocklist.yourdomain SSLEngine on SSLCertificateFile /etc/letsencrypt/live/blocklist.yourdomain/fullchain.pem SSLCertificateKeyFile /etc/letsencrypt/live/blocklist.yourdomain/privkey.pem SSLVerifyClient require SSLVerifyDepth 2 SSLCACertificateFile /etc/ssl/certs/your-ca.pem ProxyPreserveHost On ProxyPass / http://127.0.0.1:7080/ ProxyPassReverse / http://127.0.0.1:7080/ </VirtualHost>
